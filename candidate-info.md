## Project Summary

This project is a full-stack Support Ticket Management System for internal users. It allows users to create, view, update, search, filter, comment on, and progress support tickets through an enforced status lifecycle.

The application includes:

- A React frontend
- A Node.js and Express backend API
- MongoDB persistence
- Seeded users
- Ticket and comment management
- Keyword search and status filtering
- Backend input validation and error handling
- Enforced ticket status transitions
- Integration tests for valid and invalid status transitions

The project focuses on the mandatory Core scope. Authentication and user-management UI are not included.

## Setup Summary

1. Clone the repository.
2. Install dependencies for both the client and server.
3. Create the required environment files using the provided `.env.example`.
4. Start the local MongoDB service.
5. Run the database seed script to create sample users and initial data.
6. Start the backend server.
7. Start the frontend application.
8. Run the automated tests using the command documented in the README.

Detailed setup instructions, environment variables, database initialization steps, and test commands are available in `README.md` and `database/setup-notes.md`.
